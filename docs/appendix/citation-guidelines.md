# Citation Management and APA Guidelines

This document provides detailed guidance for maintaining proper citation practices and APA format compliance throughout the Physical AI & Humanoid Robotics book.

## Importance of Proper Citations

Academic integrity requires that all sources of information, ideas, and data be properly attributed. In technical documentation and educational content:

- **Credibility**: Proper citations establish the reliability of information
- **Traceability**: Readers can verify claims and access original sources
- **Scholarly Practice**: Following established citation standards
- **Avoiding Plagiarism**: Giving credit where credit is due

## APA Format Requirements (7th Edition)

### In-Text Citations

**Single Author:**
- Parenthetical: (Smith, 2023)
- Narrative: Smith (2023) demonstrated that...

**Two Authors:**
- Parenthetical: (Smith & Jones, 2023)
- Narrative: Smith and Jones (2023) found...

**Three or More Authors:**
- Parenthetical: (Smith et al., 2023)
- Narrative: Smith et al. (2023) reported...

**Multiple Sources in One Citation:**
- (Smith, 2023; Jones, 2022; Brown et al., 2021)

**Direct Quotes:**
- Short quotes (< 40 words): (Author, Year, p. #)
- Long quotes (â‰¥ 40 words): Block format without quotation marks

### Reference List Format

**Journal Articles:**
Author, A. A., & Author, B. B. (Year). Title of article. *Title of Periodical*, volume(issue), pages. https://doi.org/xx.xxx/yyyy

**Books:**
Author, A. A. (Year). *Title of work: Capital letter also for subtitle*. Publisher. https://doi.org/xx.xxx/yyyy

**Web Pages:**
Author, A. A. (Year, Month Date). Title of webpage. *Website Name*. URL

**Technical Documentation:**
Organization Name. (Year). *Title of documentation*. URL

## Technical Documentation Citations

For ROS 2, Gazebo, Isaac ROS, and other technical frameworks:

1. **Primary Documentation**: Always cite the official documentation
2. **Version Specificity**: Include version numbers when relevant
3. **URL Access Dates**: Include access dates for web-based documentation
4. **API References**: Cite specific API versions and functions

Example:
Open Robotics. (2023). *ROS 2 Humble Hawksbill Documentation: Publisher-Subscriber Communication*. https://docs.ros.org/en/humble/Tutorials/Beginner-Client-Libraries/Writing-A-Simple-Py-Publisher-And-Subscriber.html [Accessed: Dec 2023]

## Peer-Reviewed Source Requirements

This book maintains a minimum of 50% peer-reviewed sources:

### Identifying Peer-Reviewed Sources
- Academic journals with editorial boards
- Conference proceedings from established venues (ICRA, IROS, RSS, CoRL)
- Preprints from arXiv in relevant categories
- Books published by academic presses

### Quality Indicators
- Double-blind peer review process
- Editorial board with academic credentials
- Citation metrics and impact factors
- Replication of methods and results

## Maintaining the 50% Threshold

### Source Classification System
- **Peer-reviewed (P)**: Academic journals, conference papers, theses
- **Technical Documentation (T)**: Official product documentation
- **Industry Reports (I)**: Market analysis, white papers from companies
- **Educational Resources (E)**: Textbooks, course materials

### Tracking Compliance
- Regular audits of the reference list
- Classification tags for each source
- Automated tools to calculate peer-reviewed percentage
- Quarterly reviews of citation practices

### Integration with Writing Process
- In-text citation plugins for editors
- Automated reference list generation

- Collaboration features for multiple authors

## Plagiarism Prevention

### Self-Plagiarism
- Properly cite your own previous work
- Distinguish between original and adapted content
- Attribute all non-original concepts and data

### Paraphrasing Guidelines
- Maintain the original meaning while changing structure
- Cite the source even when not quoting directly
- Use quotation marks for direct quotes
- Attribute ideas even when paraphrased

### Detection and Verification
- Use plagiarism detection tools during review
- Cross-reference all claims with original sources
- Verify accuracy of citations and quotations
- Maintain version control for all changes

## Quality Assurance Process

### Pre-Publication Checks
1. **Completeness Check**: Every in-text citation has a reference
2. **Format Verification**: All references follow APA 7th edition
3. **Quality Assessment**: At least 50% peer-reviewed sources
4. **Accuracy Validation**: All URLs and DOIs are accessible
5. **Consistency Review**: Uniform citation style throughout

### Post-Publication Maintenance
- Regular review of cited sources for continued relevance
- Updates when newer sources become available
- Correction of citation errors when discovered
- Addition of sources as the field evolves

## Special Considerations for Technical Content

### Code Examples
- Attribute code from external sources
- Include license information when applicable
- Cite the original implementation or algorithm
- Note modifications from original source

### Figures and Diagrams
- Obtain permission for copyrighted images
- Attribute data sources for charts and graphs
- Cite the original source of technical diagrams
- Create original figures when possible

### Performance Benchmarks
- Cite the original benchmark implementations
- Attribute specific performance results
- Include experimental conditions and parameters
- Note when reproducing or verifying results

## Compliance Monitoring

### Automated Tools
- Script to count peer-reviewed vs. other sources
- Format validation for APA compliance
- Broken link detection for URLs
- Consistency checking across documents

### Manual Review Process
- Expert review of technical citations
- Verification of complex technical claims
- Cross-checking with original sources
- Peer review of new content additions

This citation management system ensures that the Physical AI & Humanoid Robotics book maintains the highest academic standards while providing valuable technical content to readers.
